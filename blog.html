Question-1: What are the different ways to select an element in the DOM?

In JavaScript, we can select DOM elements using various methods: document.getElementById("id") retrieves a single element by its ID, while document.getElementsByClassName("class") and document.getElementsByTagName("tag") return collections of elements by class name or tag name, respectively. We can use document.getElementsByName("name") to select elements by their name attribute, often in forms. The modern document.querySelector("selector") returns the first matching element based on a CSS selector, whereas document.querySelectorAll("selector") returns a NodeList of all matches. Additionally, document.documentElement selects the <html> element, and document.body accesses the <body>. For performance, textContent is best for pure text updates, while innerHTML is useful for structured content updates.

Question-2: What is the difference between innerHTML, innerText, and textContent ?

In JavaScript, we use innerHTML, innerText, and textContent to manipulate element content, but they behave differently. innerHTML retrieves or sets the HTML inside an element, rendering any tags, making it useful for dynamic content but risky for security if handling user input. innerText retrieves or sets only the visible text, ignoring hidden elements and maintaining formatting as displayed, but it is slower since it triggers a reflow. textContent retrieves or sets all text, including hidden elements, but treats it as plain text without parsing HTML, making it faster than innerText. For performance, textContent is best for pure text updates, while innerHTML is useful for structured content updates.

Question-3: What is event delegation in the DOM?

Event delegation in the DOM is a pattern where we leverage event bubbling to handle events efficiently by attaching a single event listener to a parent element instead of multiple listeners on child elements. When an event occurs on a child, it bubbles up to the parent, allowing us to detect and handle it centrally. This approach improves performance, reduces memory usage, and dynamically handles elements added later. We achieve this by listening for events on the parent and using event.target to identify which child triggered it. For example, in a list with many items, instead of adding a click listener to each <li>, we attach one to the <ul> and determine the clicked item using event.target.

Question-4: What is event bubbling in the DOM?

Event bubbling in the DOM is a mechanism where an event triggered on a child element propagates upward through its ancestors, allowing parent elements to listen for events fired on their descendants. When an event occurs, it first executes on the target element and then "bubbles up" through its parent elements until it reaches the document object unless stopped using event.stopPropagation(). This behavior enables efficient event handling, such as event delegation, where we attach a single listener to a parent instead of multiple listeners on child elements. For example, clicking a button inside a <div> will trigger the button’s event first, then the <div>’s event, and so on up the DOM tree.

Question-5: How do you create, add, and remove elements using JavaScript?

In JavaScript, we create elements using document.createElement(), add them using methods like appendChild(), append(), prepend(), or insertBefore(), and remove them using removeChild() or remove(). To create an element, we use let div = document.createElement("div"), set its properties like div.innerText = "Hello", and append it to a parent using document.body.appendChild(div). We can also insert elements before a reference node using parent.insertBefore(newElement, referenceNode). To remove an element, we use parent.removeChild(childElement), or if we have direct access, we call element.remove(). These methods allow us to dynamically manage DOM content efficiently.
